### 1（1）两数之和

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值** *`target`* 的那 **两个** 整数，并返回它们的数组下标。你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。你可以按任意顺序返回答案。

***解题思路***：在数组中，找两个数，两值之后等于目标值。可以创建一个哈希表，Key 为数组对应的值，value 为该数对应的索引，遍历这个数组。将每个数组中的值添加到哈希表中

```
class Solution {
    public int[] twoSum(int[] nums, int target) {
        // 解题关键：在数组中，找两个数，两值之后等于目标值
        // 可以创建一个哈希表，Key 为数组对应的值，value 为该数对应的索引

        // 创建哈希表
        Map<Integer,Integer> map = new HashMap<>();
        // 数组大小
        int length = nums.length;
        // 结果数组
        int[] result = new int[2];
        for(int i = 0 ; i < length ;i++){
            if(map.containsKey(target - nums[i])){
               result[0] = i ;
               result[1] = map.get(target - nums[i]);
               return result;
            }
            // 若不存在，就将该数添加到哈希表中
            map.put(nums[i],i);
        }
        return result;
    }
}
```



### 2（49）字母异位词分组

给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

***解题思路***：题目给了一个字符串数组，将字母异位词进行分组。换一个思路，如果这两个字符串互为字母异位词，那么这两个字符串分别进行排序，得到的结果字符串必然是相同的。那么我们将字符串排序得到的结果，相同的分为一组即可。优化：***我们可以创建一个哈希表，Key 为 字符串排序结果 ，value 为符合的字符串的集合，那么我们就可以效率极高的判断该字符串数组中是否存在自己的分组了***

```
class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        // 解题关键：题目给了一个字符串数组，将字母异位词分组到一起
        // 也就是如果这两个字符串互为字母异位词，那么将字符串分别进行排序，得到的结果字符串，这两个都是相同的
        // 这样，我们将字符排序得到的结果相同的分为一组即可
        // 我们可以创建一个哈希表，Key 为 排序结果 ，value 为符合的字符串

        // 创建哈希集合
        Map<String,List<String>> map = new HashMap<>();

        // 遍历每一个字符串
        for(String str : strs){
            // 将字符串转换为字符数组
            char[] charArray = str.toCharArray();
            // 对字符数组排序
            Arrays.sort(charArray);
            // 转换回字符串作为一个每一个分组的标识
            String tag = new String(charArray);
            // 判断该标识是否存在，也就是该分组是否已有其它元素
            List<String> list = map.getOrDefault(tag,new ArrayList<String>());
            // 将当前字符串添加到该组集合中
            list.add(str);
            // 更新哈希表
            map.put(tag,list);
        }
        
        // 得到最终的哈希表之后，将所有的 value 取出来存到一个 List 中
        List<List<String>> resultList = new ArrayList<>(map.values());

        return resultList;
    }
}
```



### 3（128）最长连续序列

给定一个未排序的整数数组 `nums` ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

***解题思路***：给了一个未排序的数组，需要找出数字连续的最长数列。给出的数组中可能一个数字会有重复出现多次，求最长的序列长度，重复数字是没有必要的。所以我们可以进行去重，将数组中的所有数字转换到一个 Set 中，将遍历数组中的数字转换成遍历 Set 中的数字。同时我们维护一个变量记录最长序列长度。***关键：同时假如当前遍历到数 num ，若集合中存在 num - 1 ，那么当前这次遍历即可跳过，最长序列不会是以当前数字开头的序列***

```
class Solution {
    public int longestConsecutive(int[] nums) {
        // 解题关键：给了一个未排序的数组，需要找出数字连续的最长数列
        // 这个数组中可能会有重复的数字，若重复数较多是影响效率的，而且我们只需要最长的数列，所以得要进行去重
        // 将数组中的数字转换到一个 Set 中，遍历 Set 中的数字，同时我们定义一个变量记录最长的序列长度
        // 同时假如当前遍历到数 num ，若集合中存在 num - 1 ，那么当前这次遍历即可跳过
        // 最长序列不会出现在以当前数字开头

        // 创建一个哈希集合
        Set<Integer> set = new HashSet<>();
        // 全局最大序列长度
        int longestLength = 0;
        // 某一次遍历当前序列长度
        int length = 0;
        // 初始化这个哈希集合
        for(int i : nums){
            // 该集合不会出现重复的数字
            set.add(i);
        }
        // 遍历这个哈希集合，不断判断每一个数字为开头的序列长度，有一些数字是可以跳过的，维护最大序列长度
        for(int num : set){
            if(set.contains(num - 1)){
                continue;
            }
            // 初始化当前序列长度
            length = 0;
            length++;
            while(set.contains(num + 1)){
                // 若存在，长度++ ，num++
                length++;
                num++;
            }
            // 维护最大序列长度
            longestLength = Math.max(length,longestLength);
        }
        // 返回
        return longestLength;
    }
}
```



### 4 （283）移动零

给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。

**请注意** ，必须在不复制数组的情况下原地对数组进行操作。

**解题思路**：需要保持非零元素的相对顺序，我们可以定义两个指针，左指针用来标记即将需要进行交换的位置，右指针遍历整个数组，若右指针对应的数非 0 就与左指针对应的数字互换，起始两个指针都指向索引 0 的位置（如果数组一开始都是非 0，那么左右指针一同向前；当遇到为 0 时，左指针不动，右指针继续右移，找下一个非 0 的数与之进行交换，左右指针继续移动，这样就确保了非零元素的相对顺序）因为需要把 0 都移到后面，同时非 0 数相对位置不可改变，那么我们可以定义两个指针，一个左指针指向的是 0 的位置，一个右指针一直向右遍历，当右指针遇到非 0 时，左右指针对应的值互换，此时，左指针对应的不是 0 了，右指针对应的是 0 了，然后左、右指针都向右走一步，也就是整体相当于，左右指针维护了一个滑动窗口，这个窗口里面都是 0 ，左指针的位置就是首 0 ，右指针 - 1 的位置是尾 0 ，右指针对应的位置，若不是 0 ，就与左指针对应的值交换位置，此时就相当于，窗口右移动了，直至右指针右移到越过数组

```
class Solution {
    public void moveZeroes(int[] nums) {
        // 需要保持非零元素的相对顺序
        // 我们可以定义两个指针，左指针用来标记需要进行交换的位置，右指针用来遍历整个数组，若右指针对应的数非 0 就与左指针对应的数字互换
        // 因为需要把 0 都移到后面，同时非 0 数相对位置不可改变
        // 那么我们可以定义两个指针，一个左指针指向的是 0 的位置，一个右指针一直向右遍历，当右指针遇到非 0 时，
        // 左右指针对应的值互换，此时，左指针对应的不是 0 了，右指针对应的是 0 了
        // 然后左、右指针都向右走一步，也就是整体相当于，左右指针维护了一个滑动窗口，这个窗口里面都是 0 
        // 左指针的位置就是首 0 ，右指针 - 1 的位置是尾 0 
        // 右指针对应的位置，若不是 0 ，就与左指针对应的值交换位置，此时就相当于，窗口右移动了
        // 直至右指针右移到越过数组

        // 起始两个指针都指向索引 0 的位置
        int left = 0;
        int right = 0 ;
        while(right < nums.length){
            // 右指针非 0 就进行互换，互换完之后左指针右移
            if(nums[right] != 0){
                swap(nums,left,right);
                left++;
            }
            // 每一次右指针都右移
            right++;
        }
        // 如果数组一开始都是正数，那么左右指针一同向前，当遇到一个为 0 时，左指针不动，右指针右移
        // 找下一个非 0 的数与之进行交换
    }

    // 交换方法
    void swap(int[] nums , int left ,int right){
        int temp = nums[left];
        nums[left] = nums[right] ;
        nums[right] = temp;
    }
}
```



### 5（11）盛最多水的容器

给定一个长度为 `n` 的整数数组 `height` 。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])` 。

找出其中的两条线，使得它们与 `x` 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

**说明：**你不能倾斜容器。

**解题思路**：需要返回容器可储存的最大水量我们可以定义左右两个指针，起始位置为整个容器的边界，不断将容器范围缩小(x在不断缩小，那么唯有高度更小的那边变为更大，容量才会更大)，定义一个全局变量，确定最大的容水量

```
class Solution {
    public int maxArea(int[] height) {
        // 关键：返回容器可储存的最大水量
        // 我们可以定义左右两个指针，起始位置为整个容器的边界
        // 不断将容器范围缩小(x在不断缩小，那么唯有高度更小的那边变为更大，容量才会更大)
        // 定义一个全局变量，确定最大的容水量
        // 需要返回容器的最大量，起始的长是固定的，随着向两边缩，长度会不断变小
        // 那么当需要缩小的时候，该选择左右两边的哪一边向里聚拢呢？
        // 容积的公式是长度*高度，因为长度是不断缩短的
        // 所以唯有高度更低的那边向里缩，当缩小之后得到的高度更高了，才可能使得体积变大
        // 维护一个最大变量，不断缩短，统计最大量是多少
        // 缩短的边界值是，左右两端点重合

        // 返回结果
        int result = 0;
        // 当前容器大小
        int current = 0;
        // 左边界
        int l = 0;
        // 右边界
        int r = height.length - 1;
        // 左右边界不断缩小，停止条件是 r>l
        while(r > l){
            current = calculate(height,l ,r);
            result = Math.max(result,current);
            if(height[r] > height[l]){
                l++;
            }else{
                r--;
            }
        }
        return result;
    }
    // 计算容器体积
    int calculate(int[] height , int left ,int right){
        return Math.min(height[left],height[right]) * (right - left);
    }
}
```



### 6（15）三数之和

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

**注意：**答案中不可以包含重复的三元组。

***解题思路***：我们可以将三数之和转化为找两数之和等于第三数的相反数。因为结果需要的是数值不是而数组索引，为了便于更好的去凑目标数，可以直接将数组进行排序（这里升序），当大于目标数；较大索引左移，小于目标数；较小索引右移。同时，为了便于去重，也就是可能一个数多次重复，若不关心依次遍历，得到的结果有可能就会多个重复了。所以当找到一组结果时，左右两边都需要进行去重，如果即将去到的索引位置与当前数字相同，就将索引移到下一个位置。

```
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        // 关键：我们可以将三数之和转化为找两数之和等于第三数的相反数
        // 结果返回的是数值不是数组坐标的索引，为了便于更好的去凑目标数，可以直接将数组进行排序（这里升序）
        // 大于目标数较大索引左移，小于目标数较小索引右移
        // 给定了一个数组，然后需要从数组里面不断遍历，选出三个数，且这三个数索引不可互为相同
        // 这三个数的和为 0 ，也就是我们当遍历到一个数字的时候，需要找出另外两个数字和为当前数的相反数
        // 给定一个无序的数组，此时如果我们直接进行遍历，就是一个暴力算法了，会有许多无效的对比
        // 因为结果需要返回的并不是数组下标，而是具体的数
        // 那么我们可以整体对数组进行排序，这样的好处是我们从索引 0 开始依次遍历右区间的所有数字
        // 右区间相当于一个窗口，左右指针，当左右指针对应的数字之和大于目标数的相反数的话，那就右指针左移动
        // 如果两数之和小于目标数字的话，就左指针右移动
        // 边界就是左指针要小于右指针
        // 1.去重1：因为是从数组0开始依次找不重复的三元组
        // 如果 1 1 2 -3 这样的话，那么开始遍历第一个元素： 1 然后右窗口是 1 2 -3，里面进行寻找，得到一个三元组 1 2 -3
        // 遍历第二个元素：1 右窗口时 2 -3 ，此时又可能得到一个重复的三元组，那么此时就得要去重了，跳过这次遍历
        // 2.去重2：当得到了一个元组之后，窗口需要缩小，肯定时同时缩小的（有序数组），那么此时左右指针对应的数字，不能再和前面的一样
        // 否则又是重复元组了

        // 结果集合
        List<List<Integer>> list = new ArrayList<>();
        // 对初始数组升序排序
        Arrays.sort(nums);
        // 数组大小
        int length = nums.length;

        for(int i = 0 ; i < length ; i++){
            // 去重，若相同就跳过该数
            if(i > 0 && nums[i] == nums[i - 1]){
                continue;
            }
            // 除去目标值之后，区间的左边界
            int left = i+1;
            // 除去目标值之后，区间的右边界
            int right = length - 1;
            // 寻找的目标值
            int target = -nums[i];
            while(right > left){
                    // 寻到目标值，添加到结果集合
                    if((nums[left] + nums[right]) == target){
                        list.add(Arrays.asList(nums[i],nums[left],nums[right]));
                    // 找到一个解之后，接下来去重检查
                    while(right > left && nums[right] == nums[right - 1]){
                        right -- ;
                    }
                    while(left < right && nums[left] == nums[left + 1]){
                        left ++ ;
                    }
                    // 去重完移动索引，左右都得动
                    right--;
                    left++;
                }else if((nums[left] + nums[right]) > target){
                    right--;
                }else{
                    left++;
                }
            }
        }
        return list;
    }
}
```



### 7（42）接雨水

给定 `n` 个非负整数表示每个宽度为 `1` 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

***解题思路***：每个位置能接的雨水量由**左右两侧的最高柱子(开区间包括自己)**中的**较矮者**决定，当前位置的接水量 = `min(左边最大高度, 右边最大高度) - 当前高度`所以我们可以创建两个数组分别，记录对应索引的左右开区间中柱子的最大高度。通过这两个数组我们就可以分别求出对应索引位置可以接多少雨水，进行累加，即可得到结果。

```
class Solution {
    public int trap(int[] height) {
        // 关键：当前索引位置能接多少雨水取决于 [0, x]和[x, length-1] 这两个区间中柱子高度分别的最大值
        // 数组长度
        int length = height.length;
        if (length == 0) return 0;

        // 定义两个数组并初始化
        // left[i]: 表示从左边到i位置（包括i）的最大高度
        int[] left = new int[length];
        // right[i]: 表示从右边到i位置（包括i）的最大高度  
        int[] right = new int[length];
        
        // 初始化边界值
        left[0] = height[0];                    // 最左边位置的最大高度就是它自身
        right[length - 1] = height[length - 1]; // 最右边位置的最大高度就是它自身
        
        // 定义返回结果
        int result = 0;

        // 构建left数组：从左往右遍历，记录每个位置左边的最大高度
        for(int i = 1; i < length; i++){
            // 当前位置左边的最大高度 = max(前一个位置的最大高度, 当前柱子的高度)
            left[i] = Math.max(left[i - 1], height[i]);
        }
        
        // 构建right数组：从右往左遍历，记录每个位置右边的最大高度
        for(int i = length - 2; i >= 0; i--){
            // 当前位置右边的最大高度 = max(后一个位置的最大高度, 当前柱子的高度)
            right[i] = Math.max(right[i + 1], height[i]);
        }
        
        // 计算每个位置能接的雨水量并累加
        for(int i = 0; i < length; i++){
            // 当前位置能接的雨水量 = min(左边最大高度, 右边最大高度) - 当前柱子高度
            // 如果当前柱子比左右边界都高，则接不到雨水（结果为0）
            result = result + (Math.min(left[i], right[i]) - height[i]);
        }

        return result;
    }
}
```



### 8（3）无重复字符的最长子串

给定一个字符串 `s` ，请你找出其中不含有重复字符的 **最长子串** 的长度。

***解题思路***：可以使用滑动窗口思想，求出每一个索引位置对应窗口的大小，定义一个全局变量，求得所有窗口的最大值。每求完一个索引位置的窗口值之后，求下一个索引窗口值时，去掉上一个索引对应的字符。

```
class Solution {
    public int lengthOfLongestSubstring(String s) {
        // 解题关键：我们可以使用滑动窗口思想，求出每一个位置对应窗口的最大值

        // 创建一个哈希集合，基于它的不可重复性，存当前窗口中的字符
        Set<Character> set = new HashSet<>();
        // 字符串的长度
        int length = s.length();
        // 当前窗口的右端点
        int right = 0;
        // 全局变量，最大窗口值
        int longest = 0;
        if(length == 0){
            return longest;
        }
        // 将起始节点添加进窗口
        set.add(s.charAt(right));

        // 起始窗口就是为 0 的，开始构建窗口，左端点就是每一个字符对应的索引
        for(int i = 0 ; i < length ; i++){
            // 因为上一次循环跳出了，因为存在重复元素，上一个索引的最大窗口值已经求出来了
            // 现在滑动窗口该往右移动了，该求这个索引的最大窗口值了
            // 把上一个索引的值移除出窗口
            if(i > 0){
                set.remove(s.charAt(i - 1));
            }

            // 判断下一个值是否可以添加进窗口
            while(right + 1 < length && !set.contains(s.charAt(right + 1))){
                set.add(s.charAt(right + 1));
                right ++ ;
            }

            // 跳出循环，证明此时即当前索引位置的最大窗口值
            longest = Math.max(longest,set.size());
        }

        return longest;
    }
}
```

### 9（438）找到字符串中所有字母异位词

给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的 **异位词** 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

***解题思路***：在 s 中找出所有 p 的异位词子串，也就是子串中对应字符出现的次数和 p 中的字符出现的次数是一样的，我们可以定义两个数组，分别记录子串以及 p 字符串各字符出现的次数，判断这两个数组是否相等，若相等即这个窗口与 p 字符串互为异位词。主要是确定边界，以及先初始化窗口。

```
class Solution {
    public List<Integer> findAnagrams(String s, String p) {
        // 解题关键：在 s 中找出所有 p 的异位词子串
        // 也就是子串中对应字符出现的次数和 p 中的字符出现的次数是一样的
        // 我们可以定义两个数组，分别记录子串以及 p 字符串各字符出现的次数判断是否相等
        // 若相等即互为异位词

        // 目标串的长度，也就是子串的长度
        int length = p.length();
        // 返回的结果集合
        List<Integer> result = new ArrayList<>();
        // 基本判断
        if(p.length() > s.length()){
            // 返回空集合
            return result;
        }

        // 创建存放目标串与子串各字符出现次数的数组(26个字符)
        int[] pCount = new int[26];
        int[] sCount = new int[26];
        // 第一个窗口先初始化
        for(int i = 0 ; i < length ; i++){
            pCount[p.charAt(i) - 'a']++;
            sCount[s.charAt(i) - 'a']++;
        }
        // 若第一个窗口就相同，将其进行添加到结果集合中
        if(Arrays.equals(pCount,sCount)){
            result.add(0);
        }

        // 最后一个窗口的起始索引为：s.length-length
        for(int i = 1 ; i <= s.length()-length ; i++ ){
            // 把前一个索引对应的字符次数去掉
            sCount[s.charAt(i - 1) - 'a'] -- ;
            // 添加新的字符进窗口中
            sCount[s.charAt(i + length - 1) - 'a'] ++;
            // 判断是否是异位词
            if(Arrays.equals(pCount,sCount)){
                result.add(i);
            }
        }

        return result;
    }
}
```



### 10（560）和为 K 的子数组

给你一个整数数组 `nums` 和一个整数 `k` ，请你统计并返回 *该数组中和为 `k` 的子数组的个数* 。子数组是数组中元素的连续非空序列。

***解题思路***：前缀和 + 哈希表优化：[j..i] 这个子数组和为 k，可以转换为 pre[i] − pre[j−1] == k，移项 pre[j−1] == pre[i] − k。我们可以统计一下 i 索引之前的前缀和等于 pre[j−1] 的索引的个数。我们通过创建一个哈希表，统计当前索引前面的各个前缀和出现的次数，我们即可以快速的获取到是否存在某个目标前缀和，以及该前缀和出现的次数。初始化哈希表时一定要添加：map.put(0,1)

```
class Solution {
    public int subarraySum(int[] nums, int k) {
        // 解题关键：前缀和 + 哈希表优化
        // [j..i] 这个子数组和为 k
        // 可以转换为 pre[i] − pre[j−1] == k
        // 移项 pre[j−1] == pre[i] − k
        // 我们可以查看一下 i 索引之前的前缀和出现的次数，也就是当前这个索引的时候，pre[j - 1] 出现的次数
        // 即是当前索引时，符合的子数组有多少个，以此类推遍历每一个索引

        // 创建哈希表
        Map<Integer,Integer> map = new HashMap<>();
        // 创建前缀和变量
        int pre = 0 ;
        // 初始化哈希表，先往哈希表中添加一个 Key 为 0 的键值对，这很重要
        map.put(0,1);
        // 次数
        int result = 0;
        // 数组长度
        int length = nums.length;
        for(int i = 0 ; i < length ; i++){
            // 计算当前位置的前缀和
            pre = pre + nums[i];
            // 判断前面是否存在子串之和等于目标值
            if(map.containsKey(pre - k)){
                result = result + map.get(pre - k);
            }
            // Map 中当前大小的前缀和出现的次数加一
            map.put(pre,map.getOrDefault(pre,0)+1);
        }

        return result;
    }
}
```



### 11（53）最大子数组和

给你一个整数数组 `nums` ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。**子数组**是数组中的一个连续部分。

***解题思路***：要求具有最大和的连续的子数组，那么我们可以通过计算当前索引的前缀和减去该索引前面索引的最小前缀和，得到的值就是最大的连续子数组和，定义一个全局变量记录最大的子数组和，每一索引的前缀和就从 0 索引开始逐渐递增累加即可。

```
class Solution {
    public int maxSubArray(int[] nums) {
        // 解题关键：要求具有最大和的连续的子数组
        // 那么我们可以通过计算当前索引的前缀和然后减去前面索引的最小前缀和
        // 得到的该值就是最大的连续子数组和

        // 定义返回值
        int result = Integer.MIN_VALUE;

        // 当前索引的前缀和
        int pre = 0;
        // 最小前缀和
        int minPre = 0;

        // 遍历该数组
        for(int i : nums){
            // 计算前缀和
            pre += i;
            result = Math.max(result,pre - minPre);
            // 判断是否需要更新最小前缀和
            minPre = Math.min(pre,minPre);
        }

        return result;
    }
}
```

### 12（56）合并区间

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [starti, endi]` 。请你合并所有重叠的区间，并返回 *一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间* 。

***解题思路***：给了一个二维数组，如果两个数组构成一个连续的区间就可以进行合并；若两个数组可以合并为一个数组，那么第二个数组的左端点必定是和第一个数组有接壤的。但是题目提供的数组是没有进行排序的，所以我们可以对这些数组，对每个数组基于左端点进行排序，相邻两个数组若是接壤就可以合并。我们可以先将一个数组添加到结果集中，然后不断的进行判断是否可以合并。

```
class Solution {
    public int[][] merge(int[][] intervals) {
        // 关键思路：给了一个二维数组，数组里面的数组，如果两个数组构成一个连续的区间就可以进行合并
        // 若两个数组可以合并为一个数组，那么第二个数组的左端点必定是和第一个数组有接壤的
        // 但是这些数组之间是没有排序的，所以我们可以对这些数组的左端点进行排序
        // 相邻两个数组若是可以接壤就可以合并

        // 对这个二维数组排序(每一个元素（数组）的左端点进行排序)
        Arrays.sort(intervals,new Comparator<int[]>(){
            public int compare(int[] num1,int[] num2){
                return num1[0] - num2[0];
            }
        });

        // 定义一个存放结果的集合
        List<int[]> resultList = new ArrayList<int[]>();
        // 初始化，将第一个元素加入到集合中，然后接下来进行遍历合并
        resultList.add(intervals[0]);
        // 含有多少个数组
        int length = intervals.length;

        for(int i = 1 ; i < length ; i++){
            // 取出结果集合中的最后一个数组
            int[] last = resultList.get(resultList.size() - 1);
            // 进行判断是否合并
            if(intervals[i][0] > last[1]){
                // 合并不了
                resultList.add(intervals[i]);
                continue;
            }
            // 需要进行合并，两个数组的右端点进行比较，选取新的右端点
            last[1] = Math.max(last[1],intervals[i][1]);
        }

        return resultList.toArray(new int[resultList.size() - 1][]);
    }
}
```



### 13（189）轮换数组

给定一个整数数组 `nums`，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。

***解题思路***：轮换数组我们可以简化成两次翻转，将数组中的位置轮换 k 个位置，如果 k = 3
结果：0 1 2 3 4 5 6 --> 4 5 6 0 1 2 3 
可以化简成：
第一次翻转：0 1 2 3 4 5 6 -> 6 5 4 3 2 1 0

第二次翻转：6 5 4 3 2 1 0 -> 4 5 6 0 1 2 3 

关键：step = k%length 需进行翻转的数的个数

通过三次翻转即可到得到结果 

```
class Solution {
    public void rotate(int[] nums, int k) {
        // 解题关键：将数组中的位置轮换 k 个位置，如果 k = 3
        // 0 1 2 3 4 5 6 --> 4 5 6 0 1 2 3 
        // 可以化简成：
        // 第一次翻转：0 1 2 3 4 5 6 -> 6 5 4 3 2 1 0
        // 第二次翻转：6 5 4 3 2 1 0 -> 4 5 6 0 1 2 3 
        // 通过三次翻转即可到得到结果 

        // 数组长度
        int length = nums.length;
        //  判断是否需要操作
        if(k % length == 0){
            return;
        }
        // 第一次翻转
        swap(nums,0,length - 1);
        // 第二次翻转
        swap(nums,0, k%length-1);
        swap(nums,k%length,length - 1);
    }
    
    void swap(int[] nums,int left ,int right){
        int temp = 0;
        while(left < right){
            temp = nums[left];
            nums[left] = nums[right];
            nums[right] = temp;
            left ++;
            right --;
        }
    }
}
```



### 14（238）除自身以外数组的乘积

给你一个整数数组 `nums`，返回 数组 `answer` ，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的乘积 。题目数据 **保证** 数组 `nums`之中任意元素的全部前缀元素和后缀的乘积都在 **32 位** 整数范围内。请 **不要使用除法，**且在 `O(n)` 时间复杂度内完成此题。

***解题思路***：题目要求，求当前索引之外其余各元素的乘积，每个索引都要进行这样的计算，那么我们可以构建两个数组，一个是左前缀乘积（不包括当前索引），一个右前缀乘积（不包括当前索引）。基于这两个数组，我们就可以很方便的求出结果。

```
class Solution {
    public int[] productExceptSelf(int[] nums) {
        // 解题关键：题目要求，求当前索引之外其余各元素的乘积
        // 每个索引都要进行这样的计算
        // 那么我们可以构建两个数组，一个是左前缀乘积（不包括当前索引），一个右前缀乘积（不包括当前索引）
        // 基于这两个数组，我们就可以很方便的求出结果
        // 数组长度
        int length = nums.length;
        // 结果数组，左前缀乘积数组，右前缀乘积数组
        int[] result = new int[length];
        int[] left = new int[length];
        int[] right = new int[length];
        // 初始化
        left[0] = 1;
        right[length - 1] = 1;
        // 构建这两个数组
        for(int i = 1 ; i < length ; i++){
            left[i] = left[i-1] * nums[i - 1];
        }
        for(int i = length - 2 ; i >= 0 ; i--){
            right[i] = right[i + 1] * nums[i + 1];
        }
        // 构建结果数组
        for(int i = 0 ; i < length ; i++){
            result[i] = left[i] * right[i];
        }
        // 返回
        return result;
    }
}
```



### 15（73）矩阵置零

给定一个 `m x n` 的矩阵，如果一个元素为 **0** ，则将其所在行和列的所有元素都设为 **0** 。请使用 **[原地](http://baike.baidu.com/item/原地算法)** 算法**。**

***解题思路***：因为要使用原地算法，我们可以使用第一行与第一列来标示当前行或者当前列是否需要全部置为 0 ，同时也需要两个标记，来标识原先未修改前的第一行与第一列是否需要全部置换为 0 ，否则我们对第一行以及第一列进行修改之后，第一行与第一列原先是否需要全部置换为 0 就未知了

```
class Solution {
    public void setZeroes(int[][] matrix) {
        // 解题关键：因为要使用原地算法
        // 我们可以使用第一行与第一列来标示当前行或者当前列是否需要全部置为 0 
        // 同时也需要两个标记来标识原先的第一行与第一列是否需要全部置换为 0 
        // 否则我们修改了原数据之后，本不应该全部置为 0 的都置为 0 了

        // 行数、列数
        int row = matrix.length;
        int col = matrix[0].length;
        // 两个标识
        boolean rowFlag = false;
        boolean colFlag =false;

        // 确定两个标识
        for(int i = 0 ; i < col ; i++){
            if(matrix[0][i] == 0){
                rowFlag = true;
                // 找出一个直接跳出循环
                break;
            }
        }
        for(int i = 0 ; i < row ; i++){
            if(matrix[i][0] == 0){
                colFlag = true;
                // 找出一个直接跳出循环
                break;
            }
        }

        // 遍历除了第一行以及第一列外的每一个元素，然后对第一行或者第一列进行修改
        for(int i = 1 ; i < row ; i++){
            for(int j = 1 ; j < col ; j++){
                if(matrix[i][j] == 0){
                    matrix[i][0] = matrix[0][j] = 0;
                }
            }
        }

        // 遍历第一行以及第一列中的每个元素，整个矩阵进行修改
        for(int i = 1; i < row ; i++){
            for(int j = 1 ; j < col ;j++){
                if(matrix[0][j] == 0 || matrix[i][0] == 0){
                    matrix[i][j] = 0;
                }
            }
        }
        // 基于那两个标识对第一行以及第一列进行xiugai
        if(rowFlag){
            for(int i = 0 ; i < col ;i++){
                matrix[0][i] = 0;
            }
        }
        if(colFlag){
            for(int i = 0 ; i < row ; i++){
                matrix[i][0] = 0;
            }
        }
    }
}
```



### 16（54）螺旋矩阵

给你一个 `m` 行 `n` 列的矩阵 `matrix` ，请按照 **顺时针螺旋顺序** ，返回矩阵中的所有元素。

***解题思路***：因为需要将矩阵顺时针螺旋输出，我们可以定义四个标识，标识上、下、左、右，各边界值，当遍历完一层之后，这四个边界标识就进行压缩，直至有边界值越界(上大于下或左大于右),也就整个矩阵遍历结束  ***注意：只有当遍历的矩阵不是一行，或者不是一列的时候，才需要遍历下边界以及左边界，否则会重复遍历（当只有一列的时候，遍历左边界的时候重复了；当只有一行的时候，遍历下边界的时候重复了），记住这个条件***

```
class Solution {
    public List<Integer> spiralOrder(int[][] matrix) {
        // 解题关键：因为需要将矩阵顺时针螺旋输出
        // 我们可以定义四个标识，标识上、下、左、右，各边界值
        // 当遍历完一层之后，这四个边界标识进行压缩
        // 直至有边界值越界(上大于下或左大于右),也就遍历结束

        // 行数
        int row = matrix.length;
        // 列数
        int col = matrix[0].length;
        int top  = 0 ;
        int bottom = row - 1 ;
        int left = 0 ;
        int right = col - 1 ;
        // 定义结果集合
        List<Integer> resultList = new ArrayList<>();

        // 进入循环，开始螺旋遍历
        while(top <= bottom && left <= right){
            // 遍历上边界
            for(int i = left ; i <= right ; i++){
                resultList.add(matrix[top][i]);
            }
            // 遍历右边界
            for(int i = top + 1 ;i <= bottom ; i++){
                resultList.add(matrix[i][right]);
            }

            // 只有当这个矩阵不是一行，或者不是一列的时候，才需要遍历下边界以及左边界
            if(left != right && top != bottom){
                // 遍历下边界
                for(int i = right - 1 ; i > left ;i--){
                    resultList.add(matrix[bottom][i]);
                }
                // 遍历左边界
                for(int i = bottom ; i > top ; i--){
                    resultList.add(matrix[i][left]);
                }
            }

            // 遍历完这一层之后，边界值均缩小
            left++;
            right--;
            top++;
            bottom--;
        }
        return resultList;
    }
}
```



### 17（48）旋转图像

给定一个 *n* × *n* 的二维矩阵 `matrix` 表示一个图像。请你将图像顺时针旋转 90 度。

你必须在**[ 原地](https://baike.baidu.com/item/原地算法)** 旋转图像，这意味着你需要直接修改输入的二维矩阵。**请不要** 使用另一个矩阵来旋转图像。

***解题思路***：需要将原来的 n*n 二维矩阵顺时针旋转 90 度，转换一下思路：对矩阵先进行水平翻转，然后再主对角线翻转，即可实现所需的效果，主要是翻转时设置的行列的边界值

```

        // 可以转换一下思路，对矩阵先进行水平翻转，然后再主对角线翻转，即可实现所需的效果

        // n 的大小
        int n = matrix.length;
        
        // 水平翻转,只需要上面一半与下面一半进行交换，若条件为全部行进行翻转的话就翻回原样了
        int temp = 0;
        for(int i = 0 ; i < n/2 ;i++){
            // 全部列
            for(int j = 0 ; j < n ; j++){

                matrix[i][j] = matrix[n - i - 1][j];
                matrix[n - i - 1][j] = temp;
            }
        }

        // 主对角线翻转，也是只将下半三角与上半三角交换即可
        for(int i = 0 ; i < n ; i++){
            // 这个下三角，每一行的列数是不一样的，不可以等于当前行的行数，否则 i = j 此时也就是对角线了
            for(int j = 0 ; j < i ; j++){
                temp = matrix[i][j];
                matrix[i][j] = matrix[j][i];
                matrix[j][i] = temp;
            }
        }
    }
}
```

### 18（160）相交链表

给你两个单链表的头节点 `headA` 和 `headB` ，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 `null` 

***解题思路***：找出两个链表相交的节点，若不相交返回 null 。这题我们很容易想到哈希集合，不过也有优化的方案。如下所示，两条链表一直下一个节点遍历即可，链表 A 独有部分长度 = a ，链表B独有部分长度 = b ，公共部分长度 = c 。

若两链表相交：一直向下遍历，那么 A 走 a+c+b ， B 走 b+c+a ，下一步他们两个都会指向相同的节点，也就是相交的节点
若两链表不相交：A 走 a+b ，B 走 b+a ，那下一步他们都会指向相同的节点，也就是 null
所以让它们一直遍历下去，当两指针指向相同的节点，跳出循环之后，判断是否是 null 即可得到答案

```
public class Solution {
    public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        // 解题关键：找出两个链表相交的节点，若不相交返回 null ，这题我们很容易想到哈希集合
        // 不过也有优化的方案，如下所示，一直遍历即可
        // 链表A独有部分长度 = a ，链表B独有部分长度 = b ，公共部分长度 = c
        // 让 A 走 a+c+b 
        // 让 B 走 b+c+a 
        // 若相交的话，那么下一步他们两个都会指向相同的节点，也就是相交的节点
        // 若不相交的话，A 走 a+b ，B 走 b+a 那下一步他们都会指向相同的节点，也就是 null
        // 所以判断当他们走到的节点相同时是否是 null 即可得到答案

        ListNode tempA = headA;
        ListNode tempB = headB;
        // 当它们不相等时就一直遍历
        while(tempA != tempB){
            tempA = tempA == null ? headB : tempA.next;
            tempB = tempB == null ? headA : tempB.next;
        }

        // 跳出循环，也就是此时 tempA 与 tempB 相等
        // 直接返回 tempA 即可
        return tempA;
    }
}
```



### 19（206）反转链表

给你单链表的头节点 `head` ，请你反转链表，并返回反转后的链表。

***解题思路***：翻转链表，也就是当前节点的下一个节点指向前节点，我们如果要改变当前节点的 next 指向，一定要记录住原始的 next 指针
因此，所以我们可以定义三个临时变量

1.记录当前节点 

2.记录当前节点的前置节点 

3.记录当前节点的原始 next 指针

翻转一个节点为例子： 1 — null    

初始状态：preNode = null ; curNode = 1 ；nextNode = null；

翻转：preNode = curNode； curNode = nextNode = null ；因为curNode = null 所以 nextNode = null ；

相当于curNode 会遍历每一个节点的位置，操作每一个节点，最终会为 null 那也就结束了，返回 preNode 即可，当 curNode 为 null ，那么 preNode 就是首节点

```
class Solution {
    public ListNode reverseList(ListNode head) {
        // 解题关键：翻转链表，也就是当前节点的下一个节点指向前节点
        // 我们改变当前节点的 next 指向后，一定要记录住原始的 next 指针
        // 所以我们可以定义三个临时变量，1.记录当前节点 2.记录当前节点的前置节点 3.记录当前节点的原始 next 指针

        // 三个临时变量
        ListNode preNode = null ;
        ListNode curNode = null ;
        ListNode nextNode = null;

        // 基本判断
        if(head == null){
            return null;
        }

        // 初始化临时变量，因为首节点那么它的前置节点即为 null
        preNode = null;
        curNode = head ;
        nextNode = curNode.next;

        // 当跳出循环之后，也就是翻转结束
        while(curNode != null){
            curNode.next = preNode;
            preNode = curNode ;
            curNode = nextNode ;
            if(curNode != null){
                nextNode = curNode.next;
            }
        }

        // 跳出循环时，preNode 的位置就是头节点
        return preNode;
    }
}
```



### 20（234）回文链表

给你一个单链表的头节点 `head` ，请你判断该链表是否为回文链表。如果是，返回 `true` ；否则，返回 `false` 。

***解题思路***：一个链表我们很难判断其是否回文，因为链表数据结果，只可以一个节点一个节点的遍历，很难同时拿到其尾节点与首节点。但如果是数组的话，我们可以快速的拿到首索引与尾索引对应的值，依次判断是否是对称，所以我们将链表转换为数组，然后进行判断

```

 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */
class Solution {
    public boolean isPalindrome(ListNode head) {
        // 解题关键：一个链表我们很难判断其是否是回文，因为我们只可以一个节点一个节点的遍历，很难同时拿到其尾节点与首节点
        // 但是如果是数组的话，我们可以快速的拿到首索引与尾索引对应的值，依次判断是否是对称
        // 所以我们将链表转换为数组，然后进行判断

        // 先将链表节点值存于 List 
        // 创建该 List 
        List<Integer> list = new ArrayList<>();
        // 构建该 List
        ListNode temp = head;
        while(temp != null){
            list.add(temp.val);
            temp = temp.next;
        }

        // 判断是否回文
        int a = 0 ;
        int b = list.size() - 1;
        while(a<b){
            if(list.get(a) != list.get(b)){
                return false;
            }
            a++;
            b--;
        }

        // 跳出循环证明是回文链表
        return true;
    }
}
```



### 21（141）环形链表

给你一个链表的头节点 `head` ，判断链表中是否有环。如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。**注意：`pos` 不作为参数进行传递** 。仅仅是为了标识链表的实际情况。如果链表中存在环 ，则返回 `true` 。 否则，返回 `false` 。

***解题思路***：我们需要判断该链表是否存在环，可以使用快慢指针，当都进入环之后，两个指针所处的位置可能是不一样的，若不一样肯定会存在一个距离，所以即可通过快慢指针，快指针永远比慢指针多走一步，那么这个距离就会不断缩小了，最终必会相遇，也就是存在环了。若不存在环，那么快指针肯定会先将遇到 null 

```
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) {
 *         val = x;
 *         next = null;
 *     }
 * }
 */
public class Solution {
    public boolean hasCycle(ListNode head) {
        // 解题关键：我们需要判断该链表是否存在环，可以使用快慢指针
        // 当都进入环之后，两个指针所处的位置可能是不一样的，但是两指针是存在一个距离的
        // 所以即可通过快慢指针，快指针永远比慢指针多走一步，那么这个距离就会不断缩小了
        // 最终必会相遇，也就是存在环了
        // 若不存在环，那么快指针肯定会先将遇到 null 

        // 基础判断
        if(head == null){
            return false;
        }
        // 判断链表是否存在两个节点，不存在也是不存在环
        if(head.next == null){
            return false;
        }
        // 定义快、慢指针
        ListNode fastNode = head.next;
        ListNode lowNode = head;

        // 进行判断快指针是否会遍历到 null 
        // 边界条件，当前 fastNode 不能为 null ，也就是当前节点不能直接就是 null 了，
        // 否则 fastNode.next 会报空指针异常
        // 同时 fastNode.next 也不能为 null 
        // 否则当 fastNode 走下一步时，fastNode = fastNode.next.next 时会有空指针异常
        while(fastNode != null && fastNode.next != null){
            // 判断快慢指针是否相遇
            if(fastNode == lowNode){
                return true;
            }
            // 还没相遇，快慢指针向下走
            fastNode = fastNode.next.next;
            lowNode = lowNode.next;
        }

        // 跳出循环，也就是该链表不是环了
        return false ;
    }
}
```



### 22（142）环形链表II

给定一个链表的头节点  `head` ，返回链表开始入环的第一个节点。 *如果链表无环，则返回 `null`。*如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（**索引从 0 开始**）。如果 `pos` 是 `-1`，则在该链表中没有环。**注意：`pos` 不作为参数进行传递**，仅仅是为了标识链表的实际情况。**不允许修改** 链表。

***解题思路***：需要求得入环点，也就是不仅需要判断是否存在环，若存在环还得要求得入环点，此时也可以使用快慢指针了。

设定慢指针每走一步，快指针走两步，快指针走的路程永远是慢指针的两倍，某一个时刻两个指针相遇了，就证明存在环，但是此时相遇的节点并不一定是入环点，这种快慢指针的设计，也就是无论何时，fast 指针走过的距离都为 slow 指针的 2 倍。
通过画图：
b+c指的是环长，b是指的是入环点到相遇点的步数，a指的是链表未成环的部分
存在一种关系： a+(n+1)b+nc=2(a+b)  2倍关系
化简得：a=c+(n−1)(b+c)
基于这个式子，我们可以得到：当两指针相遇时，定义一个指针 ptr 指向的是链表头节点，相遇之后，慢指针一直在环里面一步一步运动，ptr也一步一步运动，当慢指针与 ptr 相遇时候，该点也就是入环点了

```
public class Solution {
    public ListNode detectCycle(ListNode head) {
        // 解题关键：需要求得入环点，而不是判断该链表是否存在环
        // 此时也可以使用快慢指针了，设定慢指针每走一步，快指针走两步
        // 快指针走的路程永远是慢指针的两倍
        // 某一个时刻两个指针相遇了，就证明存在环，但是此时相遇的节点并不一定是入环点
        // 这种快慢指针的设计，也就是无论何时，fast 指针走过的距离都为 slow 指针的 2 倍，
        // 通过画图：
        // b+c指的是环长，b是指的是入环点到相遇点的步数，a指的是链表未成环的部分
        // 存在一种这个式子： a+(n+1)b+nc=2(a+b)  2倍关系
        // 化简得：a=c+(n−1)(b+c)
        // 基于这个式子，我们可以得到当两指针相遇时，定义一个指针 ptr 指向的是链表头节点
        // 相遇之后，慢指针一直在环里面一步一步运动，当慢指针与 ptr 相遇时候，该点也就是入环点了

        // 基本判断
        if(head == null){
            return null;
        }
        // 定义快慢指针
        ListNode fast = head;
        ListNode low = head;

        // 开始进行移动
        while(fast != null){
            // 慢指针移动一步
            low = low.next;
            // 快指针移动两步，但是先进行判断，避免空指针
            if(fast.next == null){
                return null;
            }
            // 快指针移动两步
            fast = fast.next.next;

            // 当快慢指针相遇，此时也就是存在环了，同时定义一个指针 ptr 指向链表头节点
            if(fast == low){
                ListNode ptr = head;
                while(ptr != low){
                    // 这两个指针继续向前
                    ptr = ptr.next;
                    low = low.next;
                }
                // 当跳出循环，此时也就是入环点了
                return ptr;
            }
        }

        // 跳出循环，也就是没有环了
        return null;
    }
}
```



### 23（21）合并两个有序链表

将两个升序链表合并为一个新的 **升序** 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。 

***解题思路***：已经提供了两个升序链表，需要将两个链表进行合并到新链表中，保持升序。我们可以每次从这两个链表中取出一个节点，两个节点进行比较，较小的节点就合并到新链表中。合并之后，合并的新链表以及比较的链表都需要后移。

```
class Solution {
    public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
        // 解题关键：已经提供了两个升序链表，需要将两个链表进行合并到新链表中，保持升序
        // 我们可以每次从这两个链表中取出一个节点，两个节点进行比较，较小的节点就合并到新链表

        // 定义一个新链表的前置节点
        ListNode preFirstNode = new ListNode();
        ListNode pre = preFirstNode;

        while(list1 != null && list2 != null){
            // 两个节点值判断，取较小节点合并，合并之后该链表头节点后移
            if(list1.val <= list2.val){
                pre.next = list1;
                list1 = list1.next;
            }else{
                pre.next = list2;
                list2 = list2.next;
            }
            pre = pre.next;
        }

        // 有可能会有一个链表是较短的，那么就将较长链表未比较部分合并到新链表中
        pre.next = list1 == null ? list2 : list1;

        // 返回头节点
        return preFirstNode.next;
    }
}
```



### 24（2）两数相加

给你两个 **非空** 的链表，表示两个非负的整数。它们每位数字都是按照 **逆序** 的方式存储的，并且每个节点只能存储 **一位** 数字。

请你将两个数相加，并以相同形式返回一个表示和的链表。你可以假设除了数字 0 之外，这两个数都不会以 0 开头。

***解题关键***：两个非空链表存储，每个节点存一个正整数，每个链表表示一个逆序的整数，需要将这两个链表表示的整数相加。结果也是以链表逆序的形式返回，我们可以从每个链表的头节点开始，对应位置相加，用一个临时变量记录当前两数相加时的进位数，传递给下一位数相加时进行进位

***注意：边界条件两个链表每一位相加，唯有当两个链表都遍历完才结束；若有一个链表先遍历结束了，那么对应位相加时用 0 顶替；同时若两链表都遍历结束了，若进位数不为 0 ，需要添加一个新的节点，这个进位也算一位数***

```
/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */
class Solution {
    public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
        // 解题关键：两个非空链表存储，每个节点存一个正整数，每个链表表示一个逆序的整数
        // 需要将这两数正常相加，也是以链表的形式返回
        // 我们可以从每个链表的头节点开始，对应位置相加，用一个临时变量记录当前两数相加时的进位数
        // 传递给下一位进行进位

        // 创建新链表
        // 新链表的前置节点
        ListNode preFirstNode = new ListNode();
        ListNode pre = preFirstNode;
        pre.next = null;
        // 所需进位的进位数
        int add = 0;

        // 有一个链表没有遍历完，都不结束
        // 只有两个链表都遍历完才结束
        while(l1 != null || l2 != null){
            // 获取即将相加的值，若当前节点为 null 的话，那么其数就为 0 
            int val1 = l1 == null ? 0 : l1.val;
            int val2 = l2 == null ? 0 : l2.val;

            // 当前节点值
            int sum = val1 + val2 + add;
            // 更新进位数
            add = sum / 10 ;
            // 创建新节点
            ListNode cur = new ListNode();
            cur.val = sum % 10 ;
            cur.next = null;
            // 连接新节点，同时 pre 保持位于尾节点
            pre.next = cur;
            pre = pre.next;
            // 如果没有遍历完就向前
            if(l1 != null){
                l1 = l1.next;
            }
            if(l2 != null){
                l2 = l2.next;
            }
        }

        // 如果遍历完还有进位数的话，那就再添加一个节点
        if(add != 0){
            ListNode node = new ListNode();
            node.val = add;
            node.next = null;
            pre.next = node;
        }

        // 返回结果
        return preFirstNode.next;
    }
}
```



### 25（19）删除链表的倒数第 N 个节点

给你一个链表，删除链表的倒数第 `n` 个结点，并且返回链表的头结点。

***解题关键***：提供了一个链表，然后我们需要删除倒数的第 n 个节点，可以先计算这个链表的长度，然后计算需要删除的节点顺数的第几个，计算出对应的索引之后，移除该位置节点即可。***因为需要设置前置节点，但是首节点的前置节点是没有的，所以可以创建一个节点来充当前置节点***

```
class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {
        // 解题关键：提供了一个链表，然后我们需要删除倒数的第 n 个节点
        // 可以先计算这个链表的长度，然后计算需要删除的节点顺数的第几个
        // 计算出对应的索引之后，移除该位置节点即可

        // 计算链表节点数
        int count = coutNode(head);
        // 计算需要删除的节点顺数的位置
        int index = count - n + 1;
        // 创建首节点的前置节点
        ListNode preFirst = new ListNode();
        preFirst.next = head;
        // 临时变量
        // 当前指向的节点
        ListNode cur = head;
        // 前置节点
        ListNode pre = preFirst;
        // 当前访问是第几个节点
        int num = 1;

        while(cur != null){
            // 若需要删除第一个节点，直接返回 cur.next
            if(index == 1){
                return cur.next;
            }
            // 找到对应位置，移除当前节点
            if(num == index){
                pre.next = cur.next;
                return head;
            }

            // 还没到对应的位置，继续遍历
            cur = cur.next;
            pre = pre.next;
            num ++ ;
        }
        
        // 返回头节点
        return head;
    }

    // 计算一个链表的节点数
    int coutNode(ListNode head){
        int count = 0;
        while(head != null){
            count++;
            head = head.next;
        }
        return count;
    }
}
```



### 26（24）两两交换链表中的节点

给你一个链表，两两交换其中相邻的节点，并返回交换后链表的头节点。你必须在不修改节点内部的值的情况下完成本题（即，只能进行节点交换）。

***解题思路：***因为是两两进行交换，我们可以每两个节点，进行构建一个新的链表，然后基于递归的方式，一步步构建出新链表

```
class Solution {
```

### 27（138）随机链表复制

给你一个长度为 `n` 的链表，每个节点包含一个额外增加的随机指针 `random` ，该指针可以指向链表中的任何节点或空节点。

构造这个链表的 **[深拷贝](https://baike.baidu.com/item/深拷贝/22785317?fr=aladdin)**。 深拷贝应该正好由 `n` 个 **全新** 节点组成，其中每个新节点的值都设为其对应的原节点的值。新节点的 `next` 指针和 `random` 指针也都应指向复制链表中的新节点，并使原链表和复制链表中的这些指针能够表示相同的链表状态。**复制链表中的指针都不应指向原链表中的节点** 。

例如，如果原链表中有 `X` 和 `Y` 两个节点，其中 `X.random --> Y` 。那么在复制链表中对应的两个节点 `x` 和 `y` ，同样有 `x.random --> y` 。

返回复制链表的头节点。

用一个由 `n` 个节点组成的链表来表示输入/输出中的链表。每个节点用一个 `[val, random_index]` 表示：

- `val`：一个表示 `Node.val` 的整数。
- `random_index`：随机指针指向的节点索引（范围从 `0` 到 `n-1`）；如果不指向任何节点，则为 `null` 。

你的代码 **只** 接受原链表的头节点 `head` 作为传入参数。

***解题思路***：我们可以先把原链表中的所有节点先创建完，新节点的 next 和 random 先不赋值，原节点与新节点的对应关系通过一个哈希表关联起来。当创建结束之后，我们再一个一个将新节点的关系，基于这个哈希表关联起来

```
class Solution {
    Map<Node,Node> map = new HashMap<>();
    // 解题关键：我们可以先把原链表中的所有节点先创建完，新节点的 next 和 random 先不赋值
    // 原节点与新节点的对应关系通过一个哈希表关联起来
    // 当创建结束之后，我们再一个一个将新节点的关系，基于这个哈希表关联起来

    public Node copyRandomList(Node head) {
        // 基本判断
        if(head == null){
            return null;
        }

        // 遍历这个链表将对应节点都创建，但是先不指定对应节点的 random 指针
        Node temp1 = head;
        while(temp1 != null){
            Node newNode = new Node(temp1.val);
            map.put(temp1,newNode);
            temp1 = temp1.next;
        }

       temp1 = head;
        while(temp1 != null){
            Node node = map.get(temp1);
            node.next = map.get(temp1.next);
            node.random = map.get(temp1.random);
            temp1 = temp1.next;
        }

        return map.get(head);
    }
}
```

### 30（104）二叉树的最大深度

给定一个二叉树 `root` ，返回其最大深度。二叉树的 **最大深度** 是指从根节点到最远叶子节点的最长路径上的节点数。

***解题思路***：我们可以通过深度优先搜索，不断计算出当前节点的左子树、右子树的最大深度，选择一个较大值，即是当前节点的最大深度了。最大深度指的是从根节点到最远叶子节点的最长路径上的节点数，也就是如果当前树就一个根节点那么最大深度就是 1 。

```
class Solution {
    public int maxDepth(TreeNode root) {
        // 解题关键：我们可以通过深度优先搜索，不断计算出当前节点的左子树、右子树的最大深度
        // 最大深度指的是从根节点到最远叶子节点的最长路径上的节点数，也就是如果当前树就一个根节点那么最大深度就是 1 
        // 选择一个较大值，即是当前节点的最大深度了
        return dfs(root);
    }

    int dfs(TreeNode root){
        // 退出递归的边界条件
        if(root == null){
            return 0 ;
        }else{
            // 计算当前根节点的左儿子的最大深度
            int leftLength = dfs(root.left);
            // 计算当前根节点的右儿子的最大深度
            int rightLength = dfs(root.right);
            // 计算当前节点的最大深度，最大深度指的是从根节点到最远叶子节点的最长路径上的节点数
            // 所以当计算当前节点的最大深度时，需要把当前节点也要加上，也就是 + 1
            int maxLength = Math.max(leftLength,rightLength) + 1;
            return maxLength;
        }
    }
}
```

###  32（101）对称二叉树

给你一个二叉树的根节点 `root` ， 检查它是否轴对称。

***解题思路***：通过深度优先搜索，不断判断当前节点的左子树与右子树是否对称
判断流程：传入两个节点，若两个节点都为 null 那就返回 true 
若两个节点有其中一个节点为 null 也就是一个为 null 一个不为 null ，那肯定是不对称的，返回 false
然后判断这两个节点的值，判断当前这个节点的左子树的左节点与右子树的右节点进行判断，再判断左子树的右节点与右子树的左节点判断

若都相同就返回 true

```
class Solution {
    public boolean isSymmetric(TreeNode root) {
        // 解题关键：通过深度优先搜索，不断判断当前节点的左子树与右子树是否对称
        // 判断流程：传入两个节点，若两个节点都为 null 那就返回 true 
        // 若两个节点有其中一个节点为 null 也就是一个为 null 一个不为 null ，那肯定是不对称的，返回 false
        // 然后判断这两个节点的值，再判断当前这个节点的左子树的左节点与右子树的右节点进行判断，再判断左子树的右节点与右子树的左节点判断
        return judge(root.left,root.right);
    }

    boolean judge(TreeNode leftNode ,TreeNode rightNode){
        // 如果左右节点都为 null
        if(leftNode == null && rightNode == null){
            return true;
        }

        // 如果有一个节点为 null 
        if(leftNode == null || rightNode == null){
            return false;
        }

        // 接下来进行判断其值以及节点的左右子树
        if(leftNode.val==rightNode.val && judge(leftNode.left,rightNode.right) && judge(leftNode.right,rightNode.left)){
            return true;
        }
        return false;
    }
}
```

### 33（543）二叉树的直径

给你一棵二叉树的根节点，返回该树的 **直径** 。二叉树的 **直径** 是指树中任意两个节点之间最长路径的 **长度** 。这条路径可能经过也可能不经过根节点 `root` 。

两节点之间路径的 **长度** 由它们之间边数表示。

***解题思路***：二叉树的直径是指树中任意两个节点之间最长路径的长度，这条路径可能经过也可能不经过根节点 root 。

***二叉树的最大深度与二叉树的直径，这两题得要细细区分一下，有异曲同工之妙***
因为不一定经过根节点，那么每个节点都有可能是最大路径的根节点，维护一个整棵树路径最大值变量
通过递归可以求出每个节点的最长路径（左子树最长路径的子节点数 + 右子树最长路径的子节点数）

不断维护这个最大值变量，其实某一个节点的最长路径就是，该左子树中最深那条路径的子节点有多少个

```
class Solution {
    // 最大值变量
    int maxLength = 0;

    public int diameterOfBinaryTree(TreeNode root) {
        // 解题关键：二叉树的直径是指树中任意两个节点之间最长路径的长度，这条路径可能经过也可能不经过根节点 root 
        // 二叉树的最大深度与二叉树的直径，这两题得要细细区分一下，有异曲同工之妙
        // 因为不一定经过根节点，那么每个节点都有可能是最大路径的根节点，维护一个整棵树路径最大值变量
        // 通过递归求出每个节点的最长路径，然后维护这个最大值变量
        // 其实某一个节点的最长路径就是，该左子树中最深那条路径的子节点有多少个
        dfs(root);
        return maxLength;
    }

    // 递归算法
    int dfs(TreeNode root){
        // 退出条件
        if(root == null){
            return 0;
        }

        // 左子树的最长路径
        int leftLength = dfs(root.left);
        // 右子树的最长路径
        int rightLength = dfs(root.right);
        // 当前节点的最长路径
        maxLength = Math.max(maxLength,leftLength + rightLength);
        return Math.max(leftLength,rightLength) + 1;
    }
}
```



### 34（102）二叉树的层序遍历

给你二叉树的根节点 `root` ，返回其节点值的 **层序遍历** 。 （即逐层地，从左到右访问所有节点）。

***解题思路***：层序遍历其实就是广度优先搜索，队列实现广度优先搜索，基于队列的先入先出特性。

首先将根节点入队，出队的时候，同时将其左右节点入队。遍历结束的条件，队列为空。同时因为要每一层需要分别的集合存储，那么当出队前先获取队列当前数的大小。这个数也就是当前这一层具有的节点数，遍历完这一层之后，然后再创建一个新的 List 存下一层的数。

```
class Solution {
    public List<List<Integer>> levelOrder(TreeNode root) {
        // 解题关键：层序遍历其实就是广度优先搜索，队列实现广度优先搜索，基于队列的先入先出特性
        // 首先将根节点入队，出队的时候，同时将其左右节点入队
        // 遍历结束的条件，队列为空
        // 同时因为要每一层需要分别的集合存储，那么当出队前先获取队列当前数的大小
        // 这个数也就是当前这一层具有的节点数

        // 结果集合
        List<List<Integer>> resultList = new ArrayList<>();

        // 基本判断
        if(root == null){
            return resultList;
        }

        // 定义一个队列
        Queue<TreeNode> queue = new LinkedList<>();
        // 先将根节点入队
        queue.offer(root);

        // 当队列不为空的时候，继续执行出队入队操作
        while(!queue.isEmpty()){
            // 因为每一层需要一个 List 进行存储
            int size = queue.size();
            // 创建一个 List 存当前层所具有的节点数
            List<Integer> list = new ArrayList<>();
            for(int i = 0 ; i < size ; i++){
                // 出队，同时将子节点入队
                TreeNode node = queue.poll();
                if(node.left != null){
                    queue.offer(node.left);
                }
                if(node.right != null){
                    queue.offer(node.right);
                }
                list.add(node.val);
            }
            resultList.add(list);
        }

        return resultList;
    }
}
```



### 35（108）将有序数组转换为二叉树

给你一个整数数组 `nums` ，其中元素已经按 **升序** 排列，请你将其转换为一棵 平衡 二叉搜索树。

***解题思路***：已经提供了有序的数组，并且需要构建一个平衡二叉树。平衡二叉树，根节点左子树的所有节点一定比右子树的所有节点要小。所以我们可以通过递归每次得到一个区间中的中间节点作为根节点

```
class Solution {
    public TreeNode sortedArrayToBST(int[] nums) {
        // 解题关键：已经提供了有序的数组，并且需要构建一个平衡二叉树
        // 平衡二叉树，根节点左子树的所有节点一定比右子树的所有节点要小
        // 所以我们可以通过递归每次得到一个区间中的中间节点作为根节点
        return sort(nums,0 , nums.length - 1);
    }

    // 寻找数组中某个区间的中间节点，也就是根节点
    TreeNode sort(int[] nums ,int left ,int right){
        if(left > right){
            return null;
        }
        // 计算区间的中间值
        int mid = left + (right - left) / 2;
        TreeNode node = new TreeNode(nums[mid]);
        node.left = sort(nums,left,mid - 1);
        node.right = sort(nums , mid + 1, right);
        // 返回新创建的节点
        return node;
    }
    
}
```



### 36（114）二叉树展开为链表

给你二叉树的根结点 `root` ，请你将它展开为一个单链表：

- 展开后的单链表应该同样使用 `TreeNode` ，其中 `right` 子指针指向链表中下一个结点，而左子指针始终为 `null` 。
- 展开后的单链表应该与二叉树 [**先序遍历**](https://baike.baidu.com/item/先序遍历/6442839?fr=aladdin) 顺序相同。

***解题思路***：因为需要把一个二叉树变为一个当链表，同时这个链表的顺序是先序遍历，可以想一下先序遍历的流程，先访问当前根节点，然后访问其左子树的根节点，然后访问左子树的左子树的根节点，最后访问完左子树的最后的右叶子节点之后，才去访问根的右子树，那么我们可以每次找到根节点的左子树的最右节点之后，将根节点的右子树连接上即可

```
class Solution {
    public void flatten(TreeNode root) {
        // 解题关键：因为需要把一个二叉树变为一个当链表，同时这个链表的顺序是先序遍历
        // 可以想一下先序遍历的流程，先访问当前根节点，然后访问其左子树的根节点
        // 然后访问左子树的左子树的根节点，最后访问完左子树的最后的右叶子节点之后，才去访问根的右子树
        // 那么我们可以每次找到根节点的左子树的最右节点之后，将根节点的右子树连接上即可

        while(root != null){
            // 找该根节点的左子树的最右叶子节点
            TreeNode temp = root.left;
            // 若该根节点有左子树那就找最右叶子节点
            if(temp != null){
                // 若该节点存在右子树,temp 才移动，保证最终 temp 处于最右叶子节点
                while(temp.right != null){
                    temp = temp.right;
                }
                // 找到之后
                temp.right = root.right;
                root.right = root.left;
                root.left = null;
            }
            // 若不存在左子树，就继续走到下一个根节点
            root = root.right;
        }
    }
}
```



### 37（236）二叉树的最近公共祖先

给定一个二叉树, 找到该树中两个指定节点的最近公共祖先。[百度百科](https://baike.baidu.com/item/最近公共祖先/8918834?fr=aladdin)中最近公共祖先的定义为：“对于有根树 T 的两个节点 p、q，最近公共祖先表示为一个节点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（**一个节点也可以是它自己的祖先**）。”

***解题思路***：需要找到两个节点的最近公共祖先，也就是找公共祖先中距离根节点最远的那一个，也就是深度最大的那一个，为了便于从叶子节点一直往上遍历其所有的根节点，我们可以将每个节点的父节点存起来，因为每个节点的值互不相同，那么 Key 就为节点值，value 为当前节点。这样从下往上遍历的时候，极高效率的获取每个节点的父节点，做完之后我们可以先将其中一个节点的所有祖先先存在一个哈希集合中，并且当前节点也可以是该节点的祖先，然后从下往上遍历另一个节点的祖先，最先出现的那个就是最近的公共祖先了

```
class Solution {
    // 创建一个哈希集合
    Map<Integer,TreeNode> map = new HashMap<>();

    // 创建一个哈希表，存其中一个节点的祖先
    Set<Integer> set = new HashSet<>();

    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        // 解题关键：需要找到两个节点的最近公共祖先，也就是找公共祖先中距离根节点最远的那一个，也就是深度最大的那一个
        // 为了便于从叶子节点一直往上遍历其所有的根节点
        // 我们可以将每个节点的父节点存起来，因为每个节点的值互不相同，那么 Key 就为节点值，value 为当前节点
        // 这样从下往上遍历的时候，极高效率的获取每个节点的父节点
        // 做完之后
        // 我们可以先将其中一个节点的所有祖先先存在一个哈希集合中，并且当前节点也可以是该节点的祖先
        // 然后从下往上遍历另一个节点的祖先，最先出现的那个就是最近的公共祖先了

        // 初始化这个哈希集合，基于递归，遍历这整棵树，然后构建每个节点的父子关系
        dfs(root);

        // 将 p 节点的祖先存于哈希集合中
        while(p != null){
            // 本节点也算其祖先
            set.add(p.val);
            TreeNode temp = map.get(p.val);
            // 当 p 等于 根节点时，此时该节点时没有父节点的
            if(temp != null){
                set.add(temp.val);
            }
            // 从下一直往上
            p = map.get(p.val);
        }

        // 找最近的公共祖先
        while(q!= null){
            // 如果包含，该节点就是公共祖先了
            if(set.contains(q.val)){
                return q;
            }
            // 否则继续向上
            q = map.get(q.val);
        }
        return null;
    }

    // 基于递归，遍历这整棵树，然后构建每个节点的父子关系
    void dfs(TreeNode root){
        if(root == null){
            return ;
        }
        if(root.left != null){
```

### 38（124）二叉树的最大路径和

二叉树中的 **路径** 被定义为一条节点序列，序列中每对相邻节点之间都存在一条边。同一个节点在一条路径序列中 **至多出现一次** 。该路径 **至少包含一个** 节点，且不一定经过根节点。**路径和** 是路径中各节点值的总和。

给你一个二叉树的根节点 `root` ，返回其 **最大路径和** 。

***解题思路***：通过深度优先遍历，基于递归，不断遍历每一个节点，遍历的过程中可以计算每个节点最大路径和，维护一个最大路径和变量

```
class Solution {
    // 最大路径和变量
    int result = Integer.MIN_VALUE;
    
    public int maxPathSum(TreeNode root) {
        // 解题关键：通过深度优先遍历，基于递归，不断遍历每一个节点，
        // 遍历的过程中可以计算每个节点最大路径和
        // 维护一个最大路径和变量

        // 通过递归计算每个节点的最大路径和，维护一个最大路径和
        dfs(root);
        return result;
    }

    int dfs(TreeNode root){
        // 边界条件
        if(root == null){
            return 0;
        }
        // 左子树的最大路径和，若左子树最大路径和为负数的话，那么相加的话，根节点的最大路径和只会更小
        // 所以若得到的左子树最大路径和为负数的话，取 0 
        int leftLength = Math.max(0,dfs(root.left));
        int rightLength = Math.max(0,dfs(root.right));

        // 计算当前节点的最大路径和，判断是否更新最大路径和最大值
        result = Math.max(result,leftLength + rightLength +root.val);

        // 返回当前节点的某一条路径（左子树这一条路径,还是右子树这一条路径）的最大值
        return Math.max(leftLength,rightLength) + root.val;
    }
}
```

### 39（200）岛屿的数量

给你一个由 `'1'`（陆地）和 `'0'`（水）组成的的二维网格，请你计算网格中岛屿的数量。岛屿总是被水包围，并且每座岛屿只能由水平方向和/或竖直方向上相邻的陆地连接形成。此外，你可以假设该网格的四条边均被水包围。

***解题思路***：给你一个二维数组，若其中某个元素为 1 的话，就是陆地，相连的陆地组成岛屿，我们可以遍历数组中的每个节点，若找到一个节点为陆地的话，岛屿数量 + 1 ，但是它还会有可能连着的陆地，此时也只算是一个岛屿数，所以需要进行辐射，直至没有陆地相连为止。此时也就是该岛屿覆盖完全了，为了便于排除已经归属一块岛屿的陆地，若辐射到的，都将其设置为水即可，通过队列记录以及辐射到的陆地，若队列为空此时该陆地所属的岛屿也就统计完成了。同时因为是二维数组，为了便于标记数组中的对应的位置可以用一个数来标识，使用一种算法：int code = row * cols + col;

```
class Solution {
    public int numIslands(char[][] grid) {
        // 解题关键：给你一个二维数组，若其中某个元素为 1 的话，就是陆地，相连的陆地组成岛屿
        // 我们可以遍历数组中的每个节点，若找到一个节点为陆地的话，岛屿数量 + 1 
        // 但是它还会有可能连着的陆地，此时也只算是一个岛屿数，所以需要进行辐射，直至没有陆地相连为止
        // 此时也就是该岛屿覆盖完全了，为了便于排除已经归属一块岛屿的陆地，若辐射到的，都将其设置为水即可
        // 通过队列记录以及辐射到的陆地，若队列为空此时该陆地所属的岛屿也就统计完成了
        // 同时因为是二维数组，为了便于标记数组中的对应的位置可以用一个数来标识
        // 使用一种算法：int code = row * cols + col;

        // 维护一个变量记录岛屿的数量
        int total = 0;
        // 创建一个队列
        Queue<Integer> queue = new LinkedList<>();
        // 行数
        int rows = grid.length;
        int cols = grid[0].length;

        // 遍历数组中的每一个位置
        for(int i = 0 ; i < rows ; i++){
            for(int j = 0 ; j < cols ; j++){
                if(grid[i][j] == '1'){
                    // 岛屿数 + 1
                    total++;
                    // 同时将当前陆地标记为 '0' 标识已被访问过了
                    grid[i][j] = '0';
                    // 还需要进行辐射，也就是该陆地相连的陆地也算同一个岛屿，需要将相连的陆地也标记为 '0'
                    // 为了便于辐射可以基于队列很好的实现
                    queue.offer(i*cols+j);

                    // 对这个队列进行处理，出队时，那么相邻的陆地需要入队
                    while(!queue.isEmpty()){
                        int code = queue.poll();
                        // 计算行标、列标
                        int row = code / cols;
                        int col = code % cols;
                        // 进行辐射
                        if(row + 1 >= 0 && row + 1 < rows){
                            if(grid[row + 1][col] == '1'){
                                grid[row + 1][col] = '0';
                                queue.offer((row+1)*cols+col);
                            }
                        }
                        if(row - 1 >= 0 && row - 1 < rows){
                            if(grid[row - 1][col] == '1'){
                                grid[row - 1][col] = '0';
                                queue.offer((row-1)*cols+col);
                            }
                        }
                        if(col + 1 >= 0 && col + 1 < cols){
                            if(grid[row][col + 1] == '1'){
                                grid[row][col + 1] = '0';
                                queue.offer((row)*cols+(col + 1));
                            }
                        }
                        if(col - 1 >= 0 && col - 1 < cols){
                            if(grid[row][col - 1] == '1'){
                                grid[row][col - 1] = '0';
                                queue.offer((row)*cols+(col - 1));
                            }
                        }
                    }
                }
            }
        }
        return total;
    }
}
```

### 40（994）腐烂的橘子

在给定的 `m x n` 网格 `grid` 中，每个单元格可以有以下三个值之一：

- 值 `0` 代表空单元格；
- 值 `1` 代表新鲜橘子；
- 值 `2` 代表腐烂的橘子。

每分钟，腐烂的橘子 **周围 4 个方向上相邻** 的新鲜橘子都会腐烂。

返回 *直到单元格中没有新鲜橘子为止所必须经过的最小分钟数。如果不可能，返回 `-1`* 。

***解题思路***：通过广度优先算法，初始先将腐烂的橘子入队，也统计新鲜橘子有多少个。出队时，先统计队列中有多少个腐烂的橘子，然后依次出队，出队的橘子同时会腐烂附近的橘子，也入队，同时新鲜橘子总数 - 1。出队完先前统计的腐烂橘子之后，时间 + 1 ，需要进行腐烂的条件是队列不为空，同时新鲜的橘子还存在
若队列为空了，但是新鲜橘子还存在，那么久返回 -1
若队列不为空，但是新鲜橘子没有了，也就是腐烂完了，队列中腐烂的橘子也就没有必要进行腐烂了，此时就直接返回所消耗的时间

```
class Solution {
    public int orangesRotting(int[][] grid) {
        // 关键思路：通过广度优先算法，初始先将腐烂的橘子入队，也统计新鲜橘子有多少个
        // 出队时，先统计队列中有多少个腐烂的橘子，然后依次出队，出队的橘子同时会腐烂附近的橘子，也入队，同时新鲜橘子总数 - 1
        // 出队完先前统计的腐烂橘子之后，时间 + 1 ，需要进行腐烂的条件是队列不为空，同时新鲜的橘子还存在
        // 若队列为空了，但是新鲜橘子还存在，那么久返回 -1
        // 若队列不为空，但是新鲜橘子没有了，也就是腐烂完了，队列中腐烂的橘子也就没有必要进行腐烂了
        // 此时就直接返回所消耗的时间

        // 定义一个队列，腐烂的橘子入队
        Queue<Integer> queue = new LinkedList<>();
        // 行数
        int rows = grid.length;
        // 列数
        int cols = grid[0].length;
        // 所耗费的时间数
        int total = 0 ;
        // 总的新鲜橘子个数
        int fresh = 0 ;

        // 先初始化队列，遍历第一遍数组，腐烂橘子入队
        for(int i = 0 ; i < rows ; i++){
            for(int j = 0 ; j < cols ; j++){
                // 腐烂的橘子入队
                if(grid[i][j] == 2){
                    int code = i * cols + j;
                    queue.offer(code);
                }
                // 同时统计总共有多少个新鲜的橘子
                if(grid[i][j] == 1){
                    fresh ++ ;
                }
            }
        }

        // 开始进行腐烂
        // 不为空就一直腐烂，同时如果 fresh 为 0 就直接结束了，不需要腐烂的橘子再腐烂别的橘子了
        while(!queue.isEmpty() && fresh > 0){
            int size = queue.size();
            // 初始时腐烂橘子有 size 个
            for(int i = 0 ; i < size ; i++){
                int code = queue.poll();
                int row = code / cols;
                int col = code % cols;
                // 进行腐烂
                if(row + 1 >= 0 && row + 1 < rows){
                    if(grid[row + 1][col] == 1){
                        // 腐烂需要入队
                        queue.offer((row + 1)*cols + col);
                        grid[row + 1][col] = 2;
                        fresh--;
                    }
                }
                if(row - 1 >= 0 && row - 1 < rows){
                    if(grid[row - 1][col] == 1){
                        // 腐烂需要入队
                        queue.offer((row - 1)*cols + col);
                        grid[row - 1][col] = 2;
                        fresh--;
                    }
                }
                if(col + 1 >= 0 && col + 1 < cols){
                    if(grid[row][col + 1] == 1){
                        // 腐烂需要入队
                        queue.offer(row*cols + (col + 1));
                        grid[row][col + 1] = 2;
                        fresh--;
                    }
                }
                if(col - 1 >= 0 && col - 1 < cols){
                    if(grid[row][col - 1] == 1){
                        // 腐烂需要入队
                        queue.offer(row*cols + (col - 1));
                        grid[row][col - 1] = 2;
                        fresh--;
                    }
                }
            }
            total++;
        }

        // 判断是否还存在新鲜橘子
        if(fresh != 0){
            return -1;
        }
        return total;
    }
}
```

给你一个整数数组 `nums` ，数组中的元素 **互不相同** 。返回该数组所有可能的子集（幂集）。解集 **不能** 包含重复的子集。你可以按 **任意顺序** 返回解集。

***解题思路***：我们可以递归+回溯的方式解决，一个数组的子集，一个数组有这么多元素，子集中对于每一个元素的存在与否都是有两种可能，所以一个数组的子集，个数就等于 2 的元素个数次方

```
class Solution {

    List<Integer> list = new ArrayList<>();
    // 定义一个结果集合
    List<List<Integer>> resultList = new ArrayList<>();

    public List<List<Integer>> subsets(int[] nums) {
        // 解题关键：我们可以递归+回溯的方式解决
        // 一个数组的子集，一个数组有这么多元素，子集中对于每一个元素的存在与否都是有两种可能
        // 所以一个数组的子集，个数就等于 2 的元素个数次方

        // 提供的数组包含的元素有多少个
        // 也就是每构建一个子集都需要进行（数组包含元素的个数）步判断，每一步都有两种可能，当前这一步判断是否将当前位置元素添加到子集中
        dfs(nums,0);
        return resultList;
    }

    void dfs(int[] nums , int step){
        if(step == nums.length){
            // 当判断的次数超过了元素的个数，证明该子集已经构建完成了
            // 将当前正在构建的子集加入到结果集合中，需要进行复制一份，否则每次添加的集合都指向同一个 List
            resultList.add(new ArrayList<>(list));
            // 回退到上一步
            return ;
        }

        // 当前判断先将当前位置元素添加到子集中
        list.add(nums[step]);
        dfs(nums,step + 1);
        // 回退到这里来了，然后我们将当前位置元素移除掉
        list.remove(list.size() - 1);
        dfs(nums,step + 1);
    }
}
```

### 42（35）搜索插入位置

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。请必须使用时间复杂度为 `O(log n)` 的算法。

***解题思路***：有序数组，在数组中搜索目标值，判断是否存在，若存在就返回该索引位置，不存的话返回它应该插入的位置，可以基于二分查找实现。插入的位置，默认值设置为 n ，因为二分算法中的判断逻辑是只有目标值小于等于 nums[mid] ，将插入位置的设置为 mid ，
但是如果这个目标值大于数组中的所有数，就不会触发设置插入位置值了，为了弥补这个缺陷，默认值就是插入到最后一个位置

```
class Solution {
    public int searchInsert(int[] nums, int target) {
        // 解题关键：有序数组，在数组中搜索目标值，判断是否存在，若存在就返回该索引位置
        // 不存的话返回它应该插入的位置
        // 可以基于二分查找实现

        // 数组大小
        int length = nums.length;

        // 初始化左边界、有边界
        int left = 0;
        int right = length - 1;
        // 插入的位置，默认值设置为 n
        // 因为设置的判断是只有目标值小于等于 mid 才会进行插入位置的默认赋值
        // 但是如果这个目标值大于数组中的所有数，就不会触发设置插入位置值了
        // 为了弥补这个缺陷，默认值就是插入到最后一个位置
        int insertPosition = length;

        // 二分
        while(left <= right){
            // 计算中间索引
            int mid = left + (right - left)/2;
            if(target <= nums[mid]){
                insertPosition = mid;
                right = mid - 1;
            }else{
                left = mid + 1;
            }
        }

        // 返回插入位置
        return insertPosition;  
    }
```

### 43（34）在排序数组中，查找元素的第一个和最后一个位置

给你一个按照非递减顺序排列的整数数组 `nums`，和一个目标值 `target`。请你找出给定目标值在数组中的开始位置和结束位置。如果数组中不存在目标值 `target`，返回 `[-1, -1]`。你必须设计并实现时间复杂度为 `O(log n)` 的算法解决此问题。

***解题思路***：因为需要查找元素的第一个和最后一个位置，也就是可能数组中的目标值会出现多次，我们可以基于二分算法查找到目标值的位置，查找到之后，因为需要的是出现的第一次以及最后一次，需要进行两次二分，第一次二分是查找第一次，一开始找到目标之后，二分区间向左，判断前面区间还有没有出现目标值；第二次二分是查找最后一次，一开始找到目标之后，二分区间向右，判断后面区间还有没有出现目标值

```
class Solution {
    public int[] searchRange(int[] nums, int target) {
        // 解题关键：因为需要查找元素的第一个和最后一个位置，也就是可能数组中的目标值会出现多次
        // 我们可以基于二分算法查找到目标值的位置，查找到之后，因为需要的是出现的第一次以及最后一次
        // 需要进行两次二分，一次二分是查找第一次，一开始找到目标之后，二分区间向左，判断前面区间还有没有出现目标值
        // 一次二分是查找最后一次，一开始找到目标之后，二分区间向右，判断后面区间还有没有出现目标值

        // 数组大小
        int length = nums.length;
        // 结果数组
        int[] resultArray = new int[]{-1,-1};
        // 每一次二分找到的目标值索引
        int index = -1;
        
        // 第一次二分，找目标值第一次出现的索引
        int left = 0 ;
        int right = length - 1;
        while(left <= right){
            int mid = left + (right - left) / 2;
            if(nums[mid] == target){
                index = mid;
                right = mid - 1;
            }else if(target < nums[mid]){
                right = mid - 1;
            }else{
                left = mid + 1;
            }
        }
        if(index != -1){
            resultArray[0] = index;
            index = -1;
        }else{
            return resultArray;
        }

        // 第二次二分，找目标值最后一次出现的索引
        left = 0 ;
        right = length - 1;
        while(left <= right){
            int mid = left + (right - left) / 2;
            if(nums[mid] == target){
                index = mid;
                left = mid + 1;
            }else if(target < nums[mid]){
                right = mid - 1;
            }else{
                left = mid + 1;
            }
        }
        if(index != -1){
            resultArray[1] = index;
        }else{
            return resultArray;
        }

        return resultArray;
    }
}
```

### 44*（33）搜索旋转排序数组

整数数组 `nums` 按升序排列，数组中的值 **互不相同** 。在传递给函数之前，`nums` 在预先未知的某个下标 `k`（`0 <= k < nums.length`）上进行了 **向左旋转**，使数组变为 `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（下标 **从 0 开始** 计数）。例如， `[0,1,2,4,5,6,7]` 下标 `3` 上向左旋转后可能变为 `[4,5,6,7,0,1,2]` 。给你 **旋转后** 的数组 `nums` 和一个整数 `target` ，如果 `nums` 中存在这个目标值 `target` ，则返回它的下标，否则返回 `-1` 。你必须设计一个时间复杂度为 `O(log n)` 的算法解决此问题。

***解题思路***：因为这个数组现在是局部有序的，所以使用正常的二分，例如：

4 5 6 7 1 2 3   目标值为 3 如果正常的二分，第一次判断之后，区间就选择为 4 5 6 了，肯定是找不到目标值的

所以我们找到中间值 mid 之后，得要判断 mid 是在哪个有序区间，就基于与 nums[0] 和 nums[length - 1] 进行判断在哪个区间了，同时还需要判断 target 在是否在有序区间里面了

```
class Solution {
    public int search(int[] nums, int target) {
        // 数组长度
       int length = nums.length;
       // 边界值
       int left = 0;
       int right = length - 1;
       while(left <= right){
        // 中间值
        int mid = left + (right - left) / 2;
        if(nums[mid] == target){
            return mid;
        }
        // 情况1：mid在左半部分，target在左半部分
        // 操作：r = mid - 1（向左收缩）
        // 情况2：mid在左半部分，target在右半部分
        // 操作：l = mid + 1（向右收缩）
        // 情况3：mid在右半部分，target在右半部分
        // 操作：l = mid + 1（向右收缩）
        // 情况4：mid在右半部分，target在左半部分
        // 操作：r = mid - 1（向左收缩）

        // 判断这个中间值是所属于哪个区间
        if(nums[mid] >= nums[0]){
            // 情况1：mid在左半部分，target在左半部分
            if(target >= nums[0] && target < nums[mid]){
                right = mid - 1;
            }else{
            // 情况2：mid在左半部分，target在右半部分
                left = mid + 1;
            }
        }
        if(nums[mid] <= nums[length - 1]){
            // 情况3：mid在右半部分，target在右半部分
            if(target <= nums[length - 1] && target > nums[mid]){
                left = mid + 1;
            }else{
            // 情况4：mid在右半部分，target在左半部分
                right = mid - 1;
            }
        }
       }
       return -1;
    }
}
```



### 45（20）有效括号

给定一个只包括 `'('`，`')'`，`'{'`，`'}'`，`'['`，`']'` 的字符串 `s` ，判断字符串是否有效。有效字符串需满足：

1. 左括号必须用相同类型的右括号闭合。
2. 左括号必须以正确的顺序闭合。
3. 每个右括号都有一个对应的相同类型的左括号。

***解题思路***：先判断这个字符串的长度是否为偶数，若不是偶数，肯定不是有效括号了，可以创建一个栈，然后遍历这个字符串，左括号入栈，当遇到右括号时，出栈，判断这个出栈的括号是否与当前括号匹配

```
class Solution {
    public boolean isValid(String s) {
        // 解题关键：先判断这个字符串的长度是否为偶数，若不是偶数，肯定不是有效括号了
        // 可以创建一个栈，然后遍历这个字符串，左括号入栈，当遇到右括号时，出栈，判断这个出栈的括号是否与当前括号匹配

        // 判断字符串的长度是否为奇数
        int length = s.length();
        if(length % 2 == 1){
            return false;
        }

        // 创建一个栈
        Deque<Character> stack = new ArrayDeque<>();
        // 创建一个哈希表，存左右括号对应关系
        Map<Character,Character> map = new HashMap<>();
        map.put('(',')');
        map.put('{','}');
        map.put('[',']');

        // 遍历这字符串，若是左括号入栈，若是右括号，栈中的元素出栈
        for(int i  = 0 ; i < length ; i++){
            if(s.charAt(i) == '(' || s.charAt(i) == '{' || s.charAt(i) == '['){
                stack.push(s.charAt(i));
            }else{
                // 先判断栈是否为空
                if(stack.isEmpty()){
                    return false;
                }
                // 若不为空，弹出，并判断是否与当前元素匹配
                char temp = stack.pop();
                if(map.get(temp) != s.charAt(i)){
                    return false;
                }
            }
        }

        // 最后判断若栈不为空
        if(!stack.isEmpty()){
            return false;
        }
        return true;
    }
}
```

给定一个数组 `prices` ，它的第 `i` 个元素 `prices[i]` 表示一支给定股票第 `i` 天的价格。你只能选择 **某一天** 买入这只股票，并选择在 **未来的某一个不同的日子** 卖出该股票。设计一个算法来计算你所能获取的最大利润。返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 `0` 。

***解题思路***：不断遍历每一个元素，若当前元素比前面的最小值要小，就更新最小值，若没有比最小值小，那么就不断计算若当前出股票的话，与前面出现的最小值相比，收入多少，进行判断是否更新收入的最大值，保证买入股票的顺序一定比卖出股票顺序先

```

        // 解题思路：不断遍历每一个元素，若当前元素比前面的最小值要小，就更新最小值
        // 若没有比最小值小，那么就不断计算若当前出股票的话，与前面出现的最小值相比，收入多少，进行判断是否更新收入的最大值
        // 保证买入股票的顺序一定比卖出股票顺序先
        
        // 定义一个变量，存目前访问的数组元素，中最小值
        int minPrice = Integer.MAX_VALUE;

        int result = 0;

            }else{

            }
        }

        // 返回最大值
        return result;
    }
}
```

假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。每次你可以爬 `1` 或 `2` 个台阶。你有多少种不同的方法可以爬到楼顶呢？

***解题思路***：要爬到第n阶，最后一步要么是从第n-1阶爬1阶上来，要么是从第n-2阶爬2阶上来,所以总方法数 = 这两种情况加起来

```
class Solution {
    public int climbStairs(int n) {
        // 解题思路：要爬到第n阶，最后一步要么是从第n-1阶爬1阶上来，要么是从第n-2阶爬2阶上来
        // 所以总方法数 = 这两种情况加起来

        // 初始化
        int p = 0 ;
        int q = 0 ;
        int r = 1 ;

        for(int i = 1 ; i <= n ; i++){
            // 前前步到目标台阶需要的步数
            p = q ;
            // 前一步到目标台阶需要的步数
            q = r ;
            // 到目标台阶数需要的步数，等于前面两步相加
            r = p + q;
        }

        // 返回目标值
        return r;
    }
}
```

### 48（118）杨辉三角

***解题思路***：杨辉三角的性质：第 n 行就有 n 个元素，第一个元素与第 n 个元素都为 1 ，第 n 行的第 x 个元素等于第 n - 1 行的第 x - 1 元素与第 x 个元素相加，基于这些性质可以解此题

```
class Solution {
    public List<List<Integer>> generate(int numRows) {
        // 解题思路：杨辉三角的性质：第 n 行就有 n 个元素，第一个元素与第 n 个元素都为 1 
        // 第 n 行的第 x 个元素等于第 n - 1 行的第 x - 1 元素与第 x 个元素相加
        // 基于这些性质可以解此题

        // 定义结果集合
        List<List<Integer>> resultList = new ArrayList<>();

        for(int i = 1 ; i <= numRows ; i++){
            List<Integer> list = new ArrayList<>();
            for(int j = 1 ; j <= i ; j++){
                if(j == 1 || j == i){
                    list.add(1);
                }else{
                    list.add(resultList.get(i - 2).get(j - 2) + resultList.get(i - 2).get(j - 1));
                }
            }

            resultList.add(list);
```

### 49（136）只出现一次的数字

给你一个 **非空** 整数数组 `nums` ，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。你必须设计并实现线性时间复杂度的算法来解决此问题，且该算法只使用常量额外空间。

***解题思路***：

1.任何数和 0 做异或运算，结果仍然是原来的数，即 a⊕0=a

2.任何数和其自身做异或运算，结果是 0，即 a⊕a=0

3.异或运算满足交换律和结合律，即 a⊕b⊕a=b⊕a⊕a=b⊕(a⊕a)=b⊕0=b ，也就是{1,2,2} 进行异或的话 0 和1 异或为 1，2 和 2 异或为0，最后 1 和 0 异或为 1

```
class Solution {
    public int singleNumber(int[] nums) {
        // 解题思路：
        // 1.任何数和 0 做异或运算，结果仍然是原来的数，即 a⊕0=a
        // 2.任何数和其自身做异或运算，结果是 0，即 a⊕a=0
        // 3.异或运算满足交换律和结合律，即 a⊕b⊕a=b⊕a⊕a=b⊕(a⊕a)=b⊕0=b
        // 也就是{1,2,2} 进行异或的话
        // 0和1异或为1，2和2异或为0，最后1和0异或为1
        int temp = 0;
        for(int num : nums){
            temp^=num;
        }
        return temp;
    }
}
```

### 51（239）滑动窗口最大值

给你一个整数数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 k 个数字。滑动窗口每次只向右移动一位。

返回 滑动窗口中的最大值 。

***解题思路***：最主要的是优先队列的定义，以及非滑动窗口中的元素从队列中剔除，并不是非滑动窗口中的元素都需要从队列中剔除

```
class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        // 给定了一个数组，还有一个窗口的大小值，需要从索引 0 开始不断滑动窗口，需要得到每个窗口中的最大值
        // 需要计算每个窗口的最大值，每滑动一个窗口，其实改变的元素就只有一个
        // 所以我们可以使用一个优先队列（堆）
        
        // 数组大小
        int size = nums.length;
        // 窗口的个数
        int winSize = size - k + 1;
        // 结果数组
        int[] result = new int [winSize];
        // 自定义一个优先队列，堆中的每一个元素是 int 数组，每一个数组中有两个元素
        // 第一个元素是题目给定数组的值，第二个元素是第一个元素的在给定数组中的索引
        PriorityQueue<int[]> pq = new PriorityQueue<int[]>(
                new Comparator<int[]>(){
                    // 这种自定义的比较器，返回的是 >0 的话，第二个就是排在第一个前面
                    public int compare(int[] a ,int[] b){
                    // 先比较数组中的第一个元素
                    return a[0] == b[0] ? b[1]-a[1]:b[0]-a[0];
                }
            }
        );
        // 先将起始窗口入队列
        for(int i = 0 ; i < k ; i++){
            pq.offer(new int[]{nums[i],i});
        }
        // 第一个窗口的最大值，即是此时优先队列中的第一个元素,pq.peak()得到的就是一个数组
        result[0] = pq.peek()[0];

        // 1 2 3 4 5 6 7
        // 此时就需要滑动窗口了
        for(int i = k ; i <= size - 1 ; i++){
            // 新元素入队
            pq.offer(new int[]{nums[i],i});
            // 判断此时的优先队列中的最大元素的索引是否是小于等于 i - k ,也就是判断这个最大值是否在窗口中
            // 其实，如果此时最大值在窗口中的话，是不会踢掉不在窗口中的元素
            // 因为队列中的元素，想踢除某一个非窗口中的元素的话是不知道该踢哪一个的，踢除堆顶那个是最简单的
            // 所以队列中的元素是会大于窗口中的元素的
            // 如果最大值不在窗口中的话，需要踢出队列，然后再获取最大值放到结果集合中
            // 这里需要不断遍历检查
            while(pq.peek()[1] <= i - k ){
                // 踢掉最大元素
                pq.poll();
            }
            result[i - k + 1] = pq.peek()[0];
        }
        // 返回结果
        return result;
    }
}
```



### 52（76）最小覆盖字串

给定两个字符串 s 和 t，长度分别是 m 和 n，返回 s 中的 最短窗口 子串，使得该子串包含 t 中的每一个字符（包括重复字符）。如果没有这样的子串，返回空字符串 ""。

测试用例保证答案唯一。

***解题思路***：最主要的是，用 128 位的数组替换哈希表，窗口的边界值时 left 和 right ，窗口时不包含 right 对应的字符的，字符长度就是 right - left 而不是 right - left + 1
当找到一个符合的窗口时，窗口缩小，需要通过 wihle 循环进行判断每当缩小一点，窗口是否符合要求，不符合就跳出继续右窗口的移动

```
class Solution {
    public String minWindow(String s, String t) {
        // 需要进行判断，如果 s 小于 t 直接返回空字符串
        if (s == null || t == null || s.length() < t.length()) {
            return "";
        }
        // 给定了两个字符串 s 和 t ，需要从 s 中搜索出包含 t 字符串每一个字符的字串，这个字串是最短的
        // 那么证明这个字串可能会包含很多其它字符，最终的目的是这个字串需要含 t 中的每个字符
        // 我们可以定义一个 128 长度的 int 数组，用来记录 t 字符串中含有的每个字符对应的个数
        // 字符的 ASCII 值对应数组中的索引
        int[] tNeed = new int[128];
        // 构造好所需要的字符数组
        for(char temp : t.toCharArray()){
            // 字符当索引，字符可以自动转换成 int 的 ASCII 值
            tNeed[temp]++;
        }
        // 同时需要知道 t 包含几个字符，也就是我们求得的最短字符串至少长度为多少
        // 每寻找到一个目标字符，该数值 - 1 ，当为 0 的时候，即搜寻结束
        int needNum = t.length();
        // 定义一个字符串长度的标记量，初始设置为最大值
        int minLength = Integer.MAX_VALUE;
        // 定义一个最短字符串的起始位置索引变量
        int start = 0 ;

        // 使用滑动窗口，定义两个左右指针标记滑动窗口位置
        int left = 0 ; 
        int right = 0 ;
        while(right < s.length()){
            // 因为是右指针不断搜寻的，所以获取右指针此时对应的字符
            char temp = s.charAt(right);
            // 判断是不是需要的字符，如果需要的话，那么 needNum-- ，查找到一个目标字符
            if(tNeed[temp] > 0){
                needNum--;
            }
            // 窗口中的字符是可以多于 t 中的字符的，后面为了省略掉多余的字符，同时又不缺失必要的字符
            // 无论怎么样，窗口中包含的字符，都需要减少
            tNeed[temp] --;
            // 右窗口尝试右移动
            right++;

            // 通过 needNum 是否等于 0 进行判断，是否搜寻出了符合要求的字串
            // 需要 while 持续判断，不断收缩，直至这个窗口不符合要求，退出循环，进行外部的右指针移动
            while(needNum == 0){
                // 计算此时窗口的长度，判断是否小于标记量
                // 此时因为上面的一步 right ++ 了但是还没判断是否在窗口的，这里整个窗口是(]这样的
                // 所以 right - left 即可
                if(minLength > right - left){
                    minLength = right - left;
                    // 记录字符串的起始索引
                    start = left ;
                }
                // 找到一个符合的字串了，然后窗口可以缩小了，缩小靠的是左指针
                // 无论如何都要 tNeed[s.charAt(left)] ++ 
                tNeed[s.charAt(left)] ++ ;
                if(tNeed[s.charAt(left)] > 0){
                    needNum ++ ;
                }
                left++;
            }
        }
        // 返回,subString 函数就是保留左，不包含右的
        return minLength == Integer.MAX_VALUE ? "" :s.substring(start,start + minLength);
    }
}
```



### 53（41）缺失的第一个正数

给你一个未排序的整数数组 nums ，请你找出其中没有出现的最小的正整数。请你实现时间复杂度为 O(n) 并且只使用常数级别额外空间的解决方案。

***解题思路***：最主要的是，给定了一个数组，数组长度是n，那么最小正整数肯定是1-n+1中。同时又需要用时间复杂度O(n)，空间复杂度常数，
那么我们可以用数的正负判断1-n中哪些出现过的，如果都出现过的话，那么最小正整数就是n+1

```
class Solution {
    public int firstMissingPositive(int[] nums) {
        // 题目的要求是：实现时间复杂度为 O(n) 并且只使用常数级别额外空间的解决方案
        // 如果没有这个外加条件，一般的解题思路是用哈希从 1 开始依次遍历，或者直接用现在这个数组进行也是从  1 开始判断
        // 第一种空间复杂度是o(n)了，第二种时间复杂度是o(n2)了
        // 都不可以解决问题，那么我们可以换一种思路，因为需要找的是无序数组中的没有出现的最小正整数
        // 那么其实数组中的那些 <= 0 的数都是可以清理掉的，无需进行判断的，那么就统一设置为 n+1 
        // 为什么设置为 n+1 呢，因为其实长度为 5 的数组，最小正整数肯定是出现在 1、2、3、4、5中，如果是连续出现了，那么最终只能是 6  
        // 之所以设置值为 n+1 ，因为长度为5的数组，槽有5个，索引0-4，我们可以用正负标记这1-5有没有出现过
        // n+1 对应的槽是索引为n的，但是这个槽是不存在的，所以有一个边界判断，如果数是大于 n 的话，是无需进行标识得
        // 那么也就是如果数组中的值有大于n的，上面例子中有大于5的数是无需判断的，那么我们可以直接设置临界值，设置n+2也是可以

        // 清理掉数组中 <= 0 的元素，因为这些不是正整数，后续我们是通过数的正负判断某一个数是否出现过的，这里直接全转换成 n+1
        // 数组长度
        int length = nums.length;
        // 清理无关元素
        for(int i = 0 ; i < length ; i++){
            if(nums[i] <= 0){
                nums[i] = length + 1;
            }
        }

        // 现在数组中的数去那是正整数了
        // 因为我们需要的是求出没有出现的最小的正整数，且空间复杂度是常熟
        // 那么我们可以用遍历数组中的每一个数，数出现的话，那么索引为（该数 - 1）的数为设置为负数，表示这个数出现过
        // 后续我们判断哪个数没出现过，就依次遍历数组中的每个元素，为正数的话，那么该（索引+1）就是没有出现过的最小正整数
        for(int i = 0 ; i < length ; i++){
            // 初始化时每个数都是正数，现在的话，每个位置的正负号当一个标识了，所以遍历每一个数的时候，获取绝对值
            // 若出现，在对应索引位置设置为负数 
            int temp = Math.abs(nums[i]);
            if(temp <= length){
                // 先获取绝对值，因为可能是负数的，之前标识过，避免出现两次后负负得正
                nums[temp - 1] = -Math.abs(nums[temp - 1]);
            }
        }

        // 开始检查，从 0 开始检查，数组中第一个不是负数的位置，那么就是缺失的最小正整数就是对应位置 +1
        // 1 2 3 4 5
        for(int i = 0 ; i < length ; i++){
            if(nums[i] > 0){
                return i+1;
            }
        }
        // 如果数组存在了 1 2 3 4 5 6 这样的连续正整数，那么缺失的最小正整数就是 7 ，也就是 n+1
        return length + 1;
    }
}
```
